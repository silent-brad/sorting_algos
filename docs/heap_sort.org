#+TITLE: Heap Sort
#+AUTHOR: Brad White

* Overview

Heap sort uses a binary heap data structure to sort an array. It first
builds a max-heap from the input, then repeatedly extracts the maximum
element and places it at the end of the array.

* Background: Binary Heap

A *binary max-heap* is a complete binary tree where every parent node is
greater than or equal to its children. When stored in an array:

- The root is at index 0.
- For a node at index =i=:
  - Left child is at =2*i + 1=
  - Right child is at =2*i + 2=
  - Parent is at =(i - 1) / 2=

* How It Works

1. *Build a max-heap*: Rearrange the array so it satisfies the max-heap
   property. Start from the last non-leaf node and "heapify" (sift down)
   each node.
2. *Extract max repeatedly*:
   a. Swap the root (largest element) with the last element of the heap.
   b. Reduce the heap size by one (the last element is now sorted).
   c. Heapify the root to restore the max-heap property.
   d. Repeat until the heap size is 1.

** The Heapify (Sift Down) Procedure

Given a node at index =i= that may violate the heap property:
1. Compare the node with its left and right children.
2. Find the largest among the three.
3. If the largest is not the node itself, swap the node with the largest
   child and recursively heapify at the child's position.

* Walkthrough

Given the array: =[4, 10, 3, 5, 1]=

** Step 1: Build Max-Heap

#+BEGIN_EXAMPLE
  Array as tree:         After heapify:
       4                     10
      / \                   /  \
    10    3       →        5    3
    / \                   / \
   5   1                 4   1

  Array: [10, 5, 3, 4, 1]
#+END_EXAMPLE

** Step 2: Extract Max

| Iteration | Swap         | Heap Portion   | Sorted Portion |
|-----------+--------------+----------------+----------------|
|         1 | 10 ↔ 1       | [5, 4, 3, 1]   | [10]           |
|         2 | 5 ↔ 1        | [4, 1, 3]      | [5, 10]        |
|         3 | 4 ↔ 1        | [3, 1]         | [4, 5, 10]     |
|         4 | 3 ↔ 1        | [1]            | [3, 4, 5, 10]  |

Final: =[1, 3, 4, 5, 10]=

* Complexity

| Case    | Time       | Space |
|---------+------------+-------|
| Best    | O(n log n) | O(1)  |
| Average | O(n log n) | O(1)  |
| Worst   | O(n log n) | O(1)  |

- Building the heap takes O(n).
- Each of the n extractions requires O(log n) heapify operations.

* Key Properties

- *Not stable*: The swaps during extraction can change relative order of
  equal elements.
- *In-place*: Requires only O(1) extra memory.
- *Not adaptive*: Always O(n log n) regardless of input.
- Guaranteed O(n log n) worst case, unlike quick sort.
