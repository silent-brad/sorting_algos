#+TITLE: Bubble Sort
#+AUTHOR: Brad White

* Overview

Bubble sort is one of the simplest sorting algorithms. It gets its name from
the way larger elements "bubble up" to the end of the array with each pass.

* How It Works

1. Start at the beginning of the array.
2. Compare each pair of adjacent elements.
3. If the left element is greater than the right element, swap them.
4. After one full pass through the array, the largest unsorted element is now
   in its correct final position at the end.
5. Repeat the process for the remaining unsorted portion of the array (one
   fewer element each time, since the end is already sorted).
6. Stop when a complete pass is made with no swaps — the array is sorted.

* Walkthrough

Given the array: =[5, 3, 8, 2, 1]=

** Pass 1
| Step | Comparison | Action | Array State       |
|------+------------+--------+-------------------|
|    1 | 5 vs 3     | Swap   | [3, 5, 8, 2, 1]  |
|    2 | 5 vs 8     | No swap| [3, 5, 8, 2, 1]  |
|    3 | 8 vs 2     | Swap   | [3, 5, 2, 8, 1]  |
|    4 | 8 vs 1     | Swap   | [3, 5, 2, 1, *8*] |

After pass 1, =8= is in its final position.

** Pass 2
| Step | Comparison | Action | Array State       |
|------+------------+--------+-------------------|
|    1 | 3 vs 5     | No swap| [3, 5, 2, 1, 8]  |
|    2 | 5 vs 2     | Swap   | [3, 2, 5, 1, 8]  |
|    3 | 5 vs 1     | Swap   | [3, 2, 1, *5*, 8] |

After pass 2, =5= is in its final position.

The process continues until no more swaps are needed.

* Complexity

| Case    | Time   | Space |
|---------+--------+-------|
| Best    | O(n)   | O(1)  |
| Average | O(n²)  | O(1)  |
| Worst   | O(n²)  | O(1)  |

- *Best case* O(n) occurs when the array is already sorted (with an
  early-termination optimization that stops when no swaps happen in a pass).
- *Worst case* O(n²) occurs when the array is sorted in reverse order.

* Key Properties

- *Stable*: Equal elements retain their relative order.
- *In-place*: Requires only O(1) extra memory.
- *Adaptive*: Can terminate early if the array becomes sorted before all
  passes complete.

* Hints for Implementation in OCaml

- You will need a mutable data structure (e.g., =Array=) since bubble sort
  relies on swapping elements in place.
- Consider using nested loops: the outer loop tracks passes, the inner loop
  performs adjacent comparisons.
- Think about how to detect that no swaps occurred in a pass so you can
  terminate early.
