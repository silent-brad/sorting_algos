#+TITLE: Selection Sort
#+AUTHOR: Brad White

* Overview

Selection sort works by repeatedly finding the minimum element from the
unsorted portion and placing it at the beginning. It divides the array into
a sorted region (left) and an unsorted region (right).

* How It Works

1. Start with the entire array as the unsorted region.
2. Find the minimum element in the unsorted region.
3. Swap it with the first element of the unsorted region.
4. The sorted region now grows by one element.
5. Repeat steps 2–4 until the unsorted region is empty.

* Walkthrough

Given the array: =[29, 10, 14, 37, 13]=

| Pass | Unsorted Region     | Min Found | Swap With | Array State            |
|------+---------------------+-----------+-----------+------------------------|
|    1 | [29, 10, 14, 37, 13] |        10 | 29        | [*10*, 29, 14, 37, 13] |
|    2 | [29, 14, 37, 13]    |        13 | 29        | [10, *13*, 14, 37, 29] |
|    3 | [14, 37, 29]        |        14 | 14        | [10, 13, *14*, 37, 29] |
|    4 | [37, 29]            |        29 | 37        | [10, 13, 14, *29*, 37] |
|    5 | [37]                |        37 | —         | [10, 13, 14, 29, *37*] |

* Complexity

| Case    | Time   | Space |
|---------+--------+-------|
| Best    | O(n²)  | O(1)  |
| Average | O(n²)  | O(1)  |
| Worst   | O(n²)  | O(1)  |

Selection sort always performs O(n²) comparisons regardless of input order
because it must scan the entire unsorted region each pass.

* Key Properties

- *Not stable*: Swapping can change the relative order of equal elements.
- *In-place*: Requires only O(1) extra memory.
- *Not adaptive*: Performance is the same regardless of input order.
- Minimizes the number of *swaps* (at most n−1), which can be beneficial
  when writes are expensive.

* Hints for Implementation in OCaml

- Use an outer loop that advances the boundary between sorted and unsorted.
- Use an inner loop to scan the unsorted region for the minimum.
- Track the index of the minimum, then swap at the end of each outer
  iteration.
